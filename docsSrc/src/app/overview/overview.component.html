The world, and computer software is no different, is evolving at an increasing pace. Everyday, a new technology is born to make things easier than with the existing, group functionality of existing or just plain-ol' solve problems of the exisintg, obviously, creating new (hopefully, better) ones. To make a wise, or educated, decision is virtually impossible. One would have to try everything there is, at that moment in time, knowing the decision is out-dated, the moment it is made.

Here, various options and combinations tools, technologies, languages, frameworks, etc., etc., will be tested to help (me) make a (less bad) decision.

Determinations

There has to be some choices made before hand, if for nothing else, to narrow the spectrum. These are:

  • No UI/UX logic on the server side;
  • The server only comunicates data and resources to the client;
  • No business logic on the client side.

Layers

A modern web solution (site or application as some call it), consists of many layers that cater for different concerns, although some technologies span more than one layer. Event the most static of the web sites has, for www.example.com, has:

  • Physical copy(ies) of files;
  • A computer reponding to (HTTP) requests.

This is just what the world-wide-web was at birth - all static. But, just for these two functionalities there an enormous variety of options.
